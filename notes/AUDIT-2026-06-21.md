## 1. Аудит данных (корпус RAG)

### Что реально в боевом RAG
- **50 уникальных PDF** → 1 630 чанков (подтверждено страницей «О системе»: 50 док / 1 630 чанков / Voyage voyage-3.5).
- Раскладка на диске: `corpus/corpus_unique/` 50 txt; `corpus/mvp50/` 47 pdf+txt; `corpus/normative/` 36 pdf+txt (mvp50 ∪ normative = 50 уникальных).
- Состав почти полностью — **нормативно-регламентные документы**: устав, положения о набсовете/исполнительном органе/комитетах, дивидендная/информационная/антикоррупционная политики, кодекс этики, управление рисками, конфликт интересов, green bonds (отчёты + верификация), часть квартальных отчётов.

### Что есть на сайте uzmrc.uz (по скраперу, 3 метода со сходимостью)
- **308 PDF-ссылок** найдено (`notes/all_file_urls.txt` 307 строк, `notes/discovery_manifest.json` 308).
- Разбивка по категориям (классификатор в `scraper.py`, эвристика по URL — приблизительная):

| Категория | На сайте | В корпусе | Пропущено |
|---|---:|---:|---:|
| Существенные факты | 174 | 0 | **174** |
| Нормативка | 35 | 34 | 1 |
| Фин.отчёты | 33 | 0 | **33** |
| Прочее | 29 | 2 | 27 |
| Облигации/акции | 14 | 0 | **14** |
| Аналитика/обзоры | 14 | 12 | 2 |
| Стратегия/бизнес-план | 9 | 0 | **9** |
| **ИТОГО** | **308** | **48*** | **260** |

\* 48 совпали по имени файла URL↔корпус; +2 demo/служебных = 50 в RAG.

### Вывод по данным
- **Покрытие сайта корпусом ≈ 16% (50 из 308).**
- Корпус — это узкий срез «нормативка». Целиком отсутствуют: существенные факты (174), фин.отчётность (33), эмиссии облигаций/акций (14), стратегия/бизнес-план (9) и регламенты/методики программы рефинансирования.
- Так как по решению **«нужно взять всё»** — корпус надо пересобрать с полного набора 308 документов.

### Риски, которые вскроются при докачке (отметить заранее)
1. **Классификатор грубый.** 174 «существенных факта» и 177 файлов в папке `/uploads/uzfiles/` размечены по ключевым словам URL — реальные типы надо проверить после скачивания (часть «фактов» может быть финотчётами/аналитикой и наоборот).
2. **404 в манифесте.** В прошлых заметках упомянуты битые ссылки в части аналитики — при `--download` нужно зафиксировать `error`-записи в `corpus/manifest.json` и дочистить.
3. **Объём ×6.** 50→308 документов. Эмбеддинги: спасает уже готовый embedding-кэш (реиндекс бесплатен) + Voyage free 200M токенов; но холодный ингест 308 PDF на пейсинге 3 RPM займёт время — закладывать пакетный прогон.
4. **Дубли ru/uz и версии по годам.** Много пар (ijroorgrus/ijroorguz, ...) и годовых/квартальных серий — дедуп по содержимому, а не только по имени.
5. **Качество OCR/парсинга.** Часть PDF может быть сканами — проверить долю пустого текста после `fitz`-извлечения (как было: 5/50 ушли в 0-чанков из-за краша воркера).

## 2. Аудит ссылок / скрапера

- `scraper.py` собирает корпус **тремя независимыми проходами** (A — витрины+пагинация, B — sitemap+leaf, C — перебор год/квартал-паттернов) и сводит по канонической ссылке — методология полноты корректная.
- Есть готовый режим **`python scraper.py --download`**: качает всё в `corpus/all/`, извлекает текст в `corpus/all_txt/`, пишет `corpus/manifest.json` (sha256, bytes, pages, chars). То есть докачка «всего» — это один прогон скрапера, отдельный код писать не нужно.
- **Перепроверить ссылки** = перезапустить discovery (сайт мог обновиться с 10 июня) → сравнить новый `all_file_urls.txt` со старым → докачать дельту.

## 3. Аудит фронта (rag-cms/frontend)

Стек: React + Vite + react-router + react-markdown. Это **админ-CMS для управления векторными базами**, а не продукт для конечного пользователя UzMRC.

### Причина «чат не нажимается» (найдено в коде)
`src/pages/RagDetail.tsx:353-356`:
```tsx
<Link to={`/rag/${rag.id}/chat`}>
  <button disabled={rag.status !== 'ready' || canAsk === false}>
    <Bot size={16} /> Чат с агентом
  </button>
</Link>
```
- Кнопка-вход в чат **молча задизейблена**, если:
  1. `rag.status !== 'ready'` — база не в статусе «ready» (индексация не завершена/упала/статус не подгрузился), или
  2. `canAsk === false` — роль не owner/admin и `member_status` ≠ active (`RagDetail.tsx:71`).
- То же у кнопки «Сравнить» (`:348-349`).
- **Нет визуального объяснения почему** (тултип/подсказка на самой кнопке отсутствует) → для пользователя это выглядит как «мёртвая кнопка = баг».
- Внутри самого чата ввод исправен: composer `RagChat.tsx:320-333`, кнопка отправки `disabled={busy || !query.trim()}` — работает.

### UX-проблемы (почему «непонятно и не юзерфрендли»)
1. **Продукт подан как dev-инструмент.** Терминология «RAG'и», «чанки», «эмбеддинги», «индексация», «status=ready», управление базами и токенами — конечному пользователю (юрист/сотрудник) это не нужно.
2. **Главный экран — список баз (`RagList`)**, а не «Задать вопрос». Основная ценность спрятана.
3. **Чат на 3-м уровне:** Дашборд → выбрать базу → RagDetail → кнопка «Чат». 2-3 клика до главной функции.
4. **Мёртвые кнопки без объяснения** (см. выше) — главный источник ощущения «сломано».
5. **Иконочный сайдбар 64px** без подписей по умолчанию — неочевидная навигация.
6. **Смешение ролей:** загрузка файлов, управление участниками, удаление базы — на одном экране с чатом. Для демо клиенту это шум.

## 3b. ВАЖНО: пропущен весь HTML-контент (не только PDF)

Дополнительная проверка свежей картой сайта (firecrawl, 2026-06-21): на uzmrc.uz **666 уникальных URL, из них ≈586 HTML-страниц с контентом прямо в тексте** и только 27 PDF в навигации (наш скрапер глубоким обходом `/uploads/` нашёл больше — 308 PDF, по PDF мы полны).

**Корневая проблема:** `scraper.py` искал документы ТОЛЬКО как `/uploads/*.pdf`. Текст HTML-страниц он не извлекал вообще. Поэтому весь содержательный HTML-пласт в корпус **не попал**:

| Раздел (HTML) | Стр. | Что это | В корпусе |
|---|---:|---|---|
| refinance-programm/**tahliliy-malumotlar** | 23 | **аналитич./технич. данные программы рефинансирования** (поквартально 2021–2026) | нет |
| mortgage-market/**ipoteka-savodxonligi** | 13 | ипотечная грамотность (методич. статьи) | нет |
| mortgage-market/**bozor-va-tahlil** | 8 | обзоры рынка ипотеки | частично |
| mortgage-market/ilmiy-va-ekspert-maqolalar | 1+ | научно-экспертные статьи | нет |
| shareholders/biznes-reja-va-strategiya | 9 | бизнес-план + стратегия | нет |
| shareholders/moliyaviy-hisobotlar | 36 | финотчёты | нет |
| shareholders/obligatsiyalar+aksiyalar-emissiyasi | 23 | эмиссии | нет |
| shareholders/muhim-faktlar | 191 | существенные факты | нет |
| press/news + ads + tenders | 162 | новости/объявления/тендеры | нет |
| about/normativ-hujjatlar | 20 | нормативка (HTML-обёртки; PDF уже взяты) | да (PDF) |

Также сайт **трёхъязычный** (uz/ru/en): есть `ru/...` и `en/...` зеркала — брали в основном uz/ru PDF.

**Ответ на вопрос «есть ли ещё технические ссылки»:** да. Технически-аналитический пласт (`refinance-programm/tahliliy-malumotlar`, обзоры рынка, ипотечная грамотность, экспертные статьи, бизнес-план/стратегия) живёт как HTML-страницы и был пропущен целиком, потому что скрапер брал только PDF. Это, вероятно, и есть «технические документы», которые ты имел в виду.

**Следствие для плана:** скрапер надо расширить — извлекать текст HTML-страниц (а не только PDF), включая ru/en-зеркала. Это отдельная доработка `scraper.py` (Pass для HTML-контента).

- **12 HTML-листингов/пагинаций** (`*-pageN`) удалены как навигационный мусор.

## 4. План (после одобрения)

### Фаза A — Данные (полный корпус)
1. Перезапустить discovery: `python scraper.py` → свежий `all_file_urls.txt`, diff со старым.
2. `python scraper.py --download` → `corpus/all/` + `corpus/all_txt/` + `manifest.json`.
3. Дедуп по содержимому (sha256), карантин 404/пустых/сканов, отчёт «скачано N / ошибок M / пустых K».
4. Пересобрать RAG на полном наборе (embedding-кэш + Voyage), проверить кросс-язык ru↔uz, обновить страницу «О системе».
5. Прогнать eval (`eval/run_eval.py`) до/после — не просел ли retrieval на расширенном корпусе.

### Фаза B — Фронт (UX для конечного пользователя)
1. **Фикс «мёртвых кнопок»:** при `status≠ready`/нет прав — показывать причину (тултип/баннер «База ещё индексируется» / «Нет доступа»), а не молчаливый disabled.
2. **Чат на главную:** для не-админа дефолтный экран = чат с готовой базой (минус навигация по «RAG'ам»).
3. **Спрятать dev-терминологию** за ролью admin: обычный пользователь видит «Задать вопрос» и «Проверить документ», без «чанков/эмбеддингов/индексации».
4. **Подписи в навигации**, хлебные крошки, пустые состояния с подсказками.
5. Проверка вживую (Playwright) + скриншоты до/после.

### Решения для согласования перед Фазой A/B
- Брать **все 308** как есть, или отфильтровать мусор (устаревшие годовые серии, дубли ru/uz держать обе?).
- Существенные факты (174 шт, новостные раскрытия) — в общий RAG или в отдельную базу/коллекцию?
- Фронт: дорабатывать текущий rag-cms или сделать тонкий «клиентский» режим поверх него.
