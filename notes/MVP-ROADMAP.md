## 2. Definition of Done (что = «идеальный MVP»)

**Функциональные критерии приёмки (из MVP-PLAN):**
1. Запрос по нормативке на **русском** → ответ с цитатой (файл + страница + точная выдержка).
2. Тот же вопрос на **узбекском** → корректный ответ.
3. Загрузка нового приказа → отчёт: что **противоречит / дублирует / дополняет / пробел**, со ссылками на нормы базы и рекомендациями.

**Операционные критерии (чтобы это был ПРОДУКТ, а не прототип):**
4. Стабильно работает без rate-limit на платном стеке; индексация и `/compare` не висят.
5. Отчёт `/compare` точный: каждый conflict/duplicate реально таков; grounding ≥ 85% на находках с нормой.
6. Развёрнуто на стабильном demo-URL, переживает рестарт, есть бэкап БД.
7. Отрепетированный 15-мин демо-сценарий по 3 критериям + запасные вопросы.
