## Полное техническое описание (MVP)

**Версия:** MVP для демонстрации клиенту
**Дата:** 18 июня 2026
**Репозиторий:** github.com/Yersultan04/uzmrc (приватный)

---

## 1. Назначение

UzMRC — система для работы с внутренними нормативными документами компании. Решает две задачи:

1. **Чат по документам** — ответы на вопросы (рус/узб) со ссылкой на источник: файл, страница, точная цитата. Кросс-языковой поиск. Защита от галлюцинаций.
2. **Сравнение документов** — разбор нового приказа/регламента на пункты и сверка каждого с действующей нормативной базой: противоречия, дубли, пробелы, дополнения. Отчёт с обоснованием, цитатой нормы и рекомендацией.

Система мультитенантная по принципу «один RAG = одна изолированная база»: своя Qdrant-коллекция, свой каталог файлов, свои чанки в Postgres. Удаление RAG каскадно убирает всё.

## 2. Архитектура и стек

| Слой | Технология |
|------|------------|
| Frontend | React 18 + Vite + TypeScript + Tailwind (SPA), nginx |
| Backend | FastAPI (Python 3.12), async SQLAlchemy, Pydantic v2 |
| Реляционная БД | PostgreSQL 16 (метаданные, чанки, FTS, кэш эмбеддингов) |
| Векторная БД | Qdrant (плотные векторы, cosine) |
| Эмбеддинги | Voyage `voyage-3.5` (1024-dim, multilingual) |
| LLM-судья / rerank | Cerebras `gpt-oss-120b` (free 1M ток/день), фолбэк OpenRouter |
| Оркестрация | Docker Compose (4 контейнера: frontend, backend, postgres, qdrant) |
| Миграции | Alembic |
| Авторизация | JWT (HS256), bcrypt, закрытая регистрация |

### Поток данных

```
Загрузка:  PDF/TXT → парсинг (PyMuPDF) → чанкинг (heading-aware) →
           эмбеддинг (Voyage, с кэшем) → Qdrant + Postgres(chunks)

Чат:       вопрос → router → ReAct-агент (поиск/цитирование) →
           grounding-проверка цитат → ответ + источники

Сравнение: документ → разбивка на пункты → по каждому: hybrid-поиск +
           rerank → LLM-судья (relation) → grounding → отчёт
```

## 3. Модуль 1 — Чат по документам (агентный RAG)

### 3.1 Гибридный поиск (retrieval)
- **Dense** — эмбеддинг запроса Voyage → поиск в Qdrant (cosine).
- **Sparse** — Postgres full-text search (`ts_rank_cd`), конфигурация языка на RAG (`russian`/`simple`).
- **Fusion** — Reciprocal Rank Fusion (RRF, `k=60`), затем топ-K судье/агенту.
- **Rerank** — LLM-rerank пула кандидатов (cross-encoder-стиль) для повышения точности.

### 3.2 Агент (ReAct + структурированный вывод)
- Цикл рассуждение→действие→наблюдение, до `AGENT_MAX_STEPS` (40 по умолчанию).
- На каждом шаге LLM возвращает строго один JSON-объект, валидируемый Pydantic (`NextStepEnvelope`, discriminated union) — структурированная генерация (SGR) через `response_format=json_object`.
- **Query Router** до цикла: regex-эвристики (идентификаторы вида `295/2025` → точный lookup; слова сравнения → декомпозиция) + LLM-fallback при низкой уверенности.
- **Инструменты агента:** `hybrid_search`, `dense_search`/`sparse_search`, `decompose_and_search` (multi-entity), `hyde_search` (редкие термины), `exact_lookup` (regex по тексту), `fetch_page`, `fetch_document`, `list_files`, `cache_fact`/`recall_fact` (scratchpad), `rerank_pool`.
- **Финал/эскалация** — отдельные виды шага: `final` (с цитатами) или `escalate` (нет ответа в базе).

### 3.3 Grounding-проверка (анти-галлюцинации)
Когда модель выдаёт финальный ответ, каждая цитата проверяется на наличие в указанном чанке:
1. Подстрока на нормализованном тексте; 2. Подстрока на агрессивно нормализованном (ловит переносы/пунктуацию, гомоглифы OCR); 3. Fuzzy-сопоставление (`SequenceMatcher`, порог ≥ 0.78).

**Уверенность ответа ограничивается долей подтверждённых цитат.** Если 2 из 3 цитат не найдены — итоговая `confidence` не выше 0.33. Если в базе нет нормы — система не отвечает и не фабрикует ссылку (см. пример 7 в наборе примеров).

### 3.4 Стриминг
Каждое событие прогона дублируется durably в `events.jsonl` и live через SSE. Реконнект поддержан (`?since=N`). SSE авторизуется одноразовым `stream_token`.

## 4. Модуль 2 — Сравнение документов

Поток: загруженный документ → разбивка на пункты (clauses) → по каждому пункту hybrid-поиск + Voyage/LLM-rerank (пул 10 → топ-5) → LLM-судья классифицирует отношение пункта к ближайшей норме базы.

**Типы отношений (relation):**
| Тип | Значение |
|-----|----------|
| `conflict` (противоречие) | пункт нельзя исполнить, не нарушив действующую норму |
| `duplicate` (дубль) | повторяет существующую норму |
| `addition` (дополнение) | расширяет/уточняет, не противореча |
| `gap` (пробел) | в базе нет нормы по теме |

По каждой находке: обоснование, цитата нормы базы (с проверкой grounding), рекомендация, уверенность. Промпт судьи усилен правилами выбора relation и few-shot примерами (борьба с занижением противоречий до «дополнений»).

**Асинхронное исполнение:** `POST /compare` возвращает `202 + run_id` и не блокирует запрос. Прогресс идёт через SSE (10% загрузка + 90% анализ). Типичный документ на 10 пунктов — ~20–25 с. Лимит — 120 пунктов за прогон.

## 5. Оптимизации и надёжность

- **Кэш эмбеддингов** (`embedding_cache`, content-addressed по `sha256(provider+model+text)`): повторная индексация того же корпуса не жжёт квоту эмбеддера — реиндекс из ~6 мин превращается в ~5 с (×77). Namespace по сигнатуре модели исключает «протекание» векторов между embedding-пространствами.
- **Token-aware суб-батчинг** эмбеддингов — один запрос не пробивает TPM-лимит тарифа.
- **OCR-fallback** (Vision-LLM) для PDF со сканами при <200 извлечённых символов.
- **Иерархия фолбэков LLM:** Cerebras (быстро, 1M ток/день) → OpenRouter (платный) → Gemini (эмбеддинги-резерв).

## 6. Безопасность (pre-demo hardening)

- JWT startup-guard (отказ старта без секрета), закрытая регистрация (только bootstrap первого админа), создание пользователей только админом.
- Rate-limit логина (10/мин/IP), `/docs` отключаемы флагом, nginx security-заголовки (CSP/HSTS и пр.).
- Per-RAG изоляция данных на всех слоях (FS/Postgres/Qdrant/API).
- Секреты только в `.env` (вне репозитория). Бэкап БД — `scripts/backup-db.sh`.

## 8. Что входит в MVP / ограничения

**Входит:** чат по базе со ссылками на источник; сравнение документа с нормами; кросс-язык ru↔uz; экспорт отчёта (.md/PDF).

**Не входит:** анализ портфеля, маскирование персональных данных, генерация презентаций, парсинг рынка/OLX.

**Ограничения:** до 120 пунктов за прогон сравнения; загрузка новых документов требует интернета и платных ключей Voyage; поиск/сравнение по уже загруженной базе — быстрые. Запуск локально через Docker.

## 10. Качество (метрики)

- Кросс-язык подтверждён: запрос «антикоррупционная политика» (рус) находит и русский `antokorrup-politika` (0.80), и узбекские `korrupsiyagaqarshi` (0.79).
