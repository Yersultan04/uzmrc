# UzMRC — ИИ-ассистент (RAG): План MVP

**Клиент:** UzMRC / АО «УзКРИ» — Узбекская ипотечная рефинансирующая компания.
**Контекст:** конкурент SimbirSoft дал предложение ($56.6K–66.5K, 3–5 мес, 7 чел.). Мы делаем MVP быстрее/дешевле/предметнее.
**Основа:** готовая платформа `rag-cms` (production-grade мультитенантная агентная RAG-платформа) → достраиваем поверх.
**Дата:** 2026-06-10 (обновлено: скоуп сужен до 2 модулей)

---

## Скоуп MVP — ровно 2 модуля

1. **RAG-ассистент по документам** (нормативка + обзоры рынка), ответы с цитатами, ru+uz.
2. ⭐ **Сравнение документов** — загрузка нового регламента → отчёт о противоречиях/дублях + рекомендации. Центральный демо-сценарий.

**ВНЕ MVP (не делаем):** анализ портфеля, маскирование PII, генерация презентаций, парсинг OLX/конкурентного рынка.

---

## База: что уже готово в rag-cms (не строим заново)

Платформа покрывает модуль 1 на ~90%:
- Гибридный поиск (dense Qdrant + sparse BM25/FTS + RRF), **per-RAG russian FTS**.
- Агент ReAct+SGR (12 инструментов, query-router, LLM-reranker).
- **Grounding-верификация цитат** — ответ = файл + страница + точная цитата, выдуманные цитаты отклоняются.
- Извлечение таблиц (PDF/XLSX → markdown + LLM-описание), Vision-OCR fallback, contextual enrichment.
- Users + JWT, мультитенантность, изоляция RAG, чат-сессии, SSE-стриминг.
- Мультипровайдер: cloud (OpenAI/Voyage) / OSS (gpt-oss/Qwen3-VL/bge-m3) / **on-prem offline** (TEI/vLLM).
- Деплой: docker-compose dev/prod/onprem.

## Модуль 2 — Сравнение документов ⭐ → основная разработка

**Флоу:**
1. Загрузка нового документа → `parse_file` (✅ PDF/xlsx/txt).
2. `chunk_pages` режет на положения/пункты (✅).
3. По каждому положению → `hybrid_search` находит топ-K релевантных норм в базе (✅).
4. 🔨 **LLM-классификатор отношения**: пара «положение нового дока ↔ норма базы» → `дубль` / `противоречие` / `дополнение` / `пробел` + обоснование.
5. 🔨 **Агрегатор отчёта**: сводка по документу — конфликты/дубли/рекомендации, каждый пункт с цитатой из базы + grounding-верификация (✅).

**Что писать:**
- `backend/app/compare/` — splitter положений, retrieval-per-clause, LLM-судья, агрегатор.
- `POST /api/rags/{id}/compare` — endpoint (переиспользует parser/chunker/retrieval).
- Pydantic-схемы отчёта + SSE-прогресс (✅ инфраструктура есть).
- Frontend `RagCompare.tsx` — загрузка дока + таблица расхождений (положение → тип → норма из базы → рекомендация).

**Переиспользуем ~80% инфраструктуры.** Новое — логика сравнения + UI. **~7–10 дней.**

## Критерии приёмки (демо)

1. Аналитик спрашивает по нормативке на ru → ответ с цитатой (файл + страница + точная цитата).
2. Тот же вопрос на узбекском → корректный ответ.
3. Загружает новый приказ → отчёт: какие пункты противоречат/дублируют существующие нормы + рекомендации, со ссылками на нормы базы.
